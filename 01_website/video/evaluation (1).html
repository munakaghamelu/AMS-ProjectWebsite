List of known bugs

ID Bug Description Priority
1 The result of the combined text file had the wrong coordinates/coordinates appeared to be unchanging. We discovered the new data we recorded was overwriting the previous coordinates. So all the coordinates seemed to be constant, rather than changing. We realised we had wrongly declared the variables as global variables, so the coordinates were always being overwritten before the new coordinates were added to the text file. High
2 Skeleton is very jittery, need to smoothen the coordinates and timestamps. Solved this by resorting to interpolation to keep the time intervals constant before merging. High
3 Loading the wrong file when playing from a selected file. Turns out the filename variable needed to be static, otherwise it would be overwritten after the scene changes. High
4 Originally the slider did not work with changing the time in the playback scene. It was discovered that the kinect text file was read and processed line by line so there was no way to go back. The solution to this was to read and save the entire file into a dictionary with its time stamps so that it could be accessed on demand. High
5 The speed of the playback does not match the actual speed correctly. This was because Update() function in unity varied in its rate so we needed to use the FixedUpdate() function and scale the speed to match with Unity's speed at 50fps. High

Critical evaluation of the project

UI/UX

We went for a simple and intuitive user interface that would be easy to follow. We found that going for a minimal look was suitable because we wanted it to be functional yet professional looking since its mostly aimed for experienced dancers and judges. We made the UI for the playback scene in a way that would allow the user to mostly focus on watching the dance and not how to use the tools. The features of being able to control the time and focus on certain joints were great additions because they enable the user to take full control and properly analyse the dance.We also decided that for phase 2, the similarities would be visualised with bars instead of just numbers to give the judges a better indication of the scale of the scores. We tried our best to clarify any complex areas such as the recording scene by adding instructions, but the one area that still may confuse people is the uploading and merging of files since it is quite technical. In the future we could perhaps have a dynamic tool that tells you what to do step by step.

Functionality

Stability

We found that our core scripts are reasonably stable because they will always behave correctly when the application is used in the right process. The unit tests we have also show that the most important functionalities work well. However, we did not have enough time to write error checking for all our programs so it is likely that there are edge cases that could break the code if the user were to use the application in an unintended way. Our cloud systems needs improvement in stability because currently the user can send any file into the cloud without much checking or replace any file that has the same name. Additionally, there is a higher chance of error in the recording scene, which seems to be the most vulnerable since it requires you to connect the Kinect, which adds more complexity to the code. Thus, even though the main functionalities perform well and are reliable, given more time there needs to be more error checking to improve the stability and reliability of the code.
